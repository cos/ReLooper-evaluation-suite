124  * <h1>Theory</h1>
125  * <h2>Overview</h2>
126  * The algorithm computes a measure of similarity between two texts based on
127  * the number and form of their common subsequences, which need not be
128  * contiguous. This method can be parametrized by specifying the subsequence
129  * length k, the penalty factor lambda, which penalizes non-contiguous matches,
130  * and optional 'lambda pruning', which takes maxLambdaExponent,
131  * <code>m</code>, as parameter. Lambda pruning causes very 'stretched'
132  * substring matches not to be counted, thus speeding up the computation. The
133  * functionality of SSK and SSK-LP is explained in the following using simple
134  * examples.
135  * 
136  * <h2>Explanation &amp; Examples</h2>
137  * for all of the following examples, we assume these parameter values: 
138  *<pre> 
139  *k=2
140  *lambda=0.5
141  *m=8 (for SSK-LP examples)
142  *</pre>
143  * 
144  * <h3>SSK</h3>
145  * 
146  * <h4>Example 1</h4>
147  * 
148  * <pre>
149  *SSK(2,"ab","axb")=0.5^5 = 0,03125
150  *</pre>
151  * There is one subsequence of the length of 2 that both strings have in
152  * common, "ab".  The result of SSK is computed by raising lambda to the power
153  * of L, where L is the length of the subsequence match in the one string plus
154  * the length of the subsequence match in the other, in our case:
155  * <pre>
156  *&nbsp;  ab    axb
157  *L= 2  +   3 = 5
158  * </pre>
159  * hence, the kernel yields 0.5^5 = 0,03125
160  *
161  * <h4>Example 2</h4>
162  * <pre>
163  *SSK(2,"ab","abb")=0.5^5 + 0.5^4 = 0,09375
164  *</pre>
165  * Here, we also have one subsequence of the length of 2 that both strings have
166  * in common, "ab".  The result of SSK is actually computed by summing over all
167  * values computed for each occurrence of a common subsequence match. In this
168  * example, there are two possible cases:
169  * <pre>
170  *ab    abb
171  *--    --  L=4
172  *--    - - L=5
173  * </pre>
174  * we have two matches, one of the length of 2+2=4, one of the length of 2+3=5, 
175  * so we get the result 0.5^5 + 0.5^4 = 0,09375.
176  *
177  * <h3>SSK-LP</h3>
178  * Without lambda pruning, the string kernel finds *all* common subsequences of
179  * the given length, whereas with lambda pruning, common subsequence matches
180  * that are too much stretched in both strings are not taken into account. It
181  * is argued that the value yielded for such a common subsequence is too low
182  * (<code>lambda ^(length[match_in_s] + length[match_in_t]</code>) . Tests have
183  * shown that a tremendous speedup can be achieved using this technique while
184  * suffering from very little quality loss. <br>
185  * Lambda pruning is parametrized by the maximum lambda exponent. It is a good
186  * idea to choose that value to be about 3 or 4 times the subsequence length as
187  * a rule of thumb. YMMV.
188  *
189  * <h4>Example 3</h4>
190  * Without lambda pruning, one common subsequence, 
191  * "AB" would be found in the following two strings. (With k=2)  
192  * <pre>
193  *SSK(2,"ab","axb")=0.5^14 = 0,00006103515625
194  *</pre>
195  * lambda pruning allows for the control of the match length. So, if m 
196  * (the maximum lambda exponent) is e.g. 8, these two strings would 
197  * yield a kernel value of 0:
198  * <pre>
199  *with lambda pruning:    SSK-LP(2,8,"AxxxxxxxxxB","AyB")= 0
200  *without lambda pruning: SSK(2,"AxxxxxxxxxB","AyB")= 0.5^14 = 0,00006103515625  
201  *</pre>
202  * This is because the exponent for lambda (=the length of the subsequence
203  * match) would be 14, which is &gt; 8. In Contrast, the next result is
204  * &gt; 0
205  *<pre>
206  *m=8
207  *SSK-LP(2,8,"AxxB","AyyB")=0.5^8 = 0,00390625
208  *</pre>
209  * because the lambda exponent would be 8, which is just accepted by lambda
210  * pruning.
211  *
212  * <h3>Normalization</h3>
213  * When the string kernel is used for its main purpose, as the kernel of a
214  * support vector machine, it is not normalized.  The normalized kernel can be
215  * switched on by -F (feature space normalization) but is much slower.  Like
216  * most unnormalized kernels, K(x,x) is not a fixed value, see the next
217  * example.
218  *
219  * <h4>Example 4</h4> 
220  *<pre>
221  *SSK(2,"ab","ab")=0.5^4 = 0.0625
222  *SSK(2,"AxxxxxxxxxB","AxxxxxxxxxB") = 12.761724710464478
223  *</pre>
224  * SSK is evaluated twice, each time for two identical strings. A good measure
225  * of similarity would produce the same value in both cases, which should  
226  * indicate the same level of similarity. The value of the normalized SSK would
227  * be 1.0 in both cases. So for the purpose of computing string similarity the
228  * normalized kernel should be used. For SVM the unnormalized kernel is usually
229  * sufficient.
230  *
231  * <h2>Complexity of SSK and SSK-LP</h2>
232  * The time complexity of this method (without lambda pruning and with an
233  * infinitely large cache) is<br> 
234  * <pre>O(k*|s|*|t|)</pre>
235  * Lambda Pruning has a complexity (without caching) of<br> 
236  * <pre>O(m*binom(m,k)^2*(|s|+n)*|t|)</pre> <br>  
237  * <pre>
238  *k...          subsequence length (ssl)
239  *s,t...        strings
240  *|s|...        length of string s
241  *binom(x,y)... binomial coefficient (x!/[(x-y)!y!])
242  *m...          maxLambdaExponent (ssl-max)
243  *</pre>
244  * 
245  * Keep in mind that execution time can increase fast for long strings 
246  * and big values for k, especially if you don't use lambda pruning.
247  * With lambda pruning, computation is usually so fast that switching
248  * on the cache leads to slower computation because of setup costs. Therefore
249  * caching is switched off for lambda pruning.
250  * <br>
251  * <br>
252  * For details and qualitative experiments about SSK, see [1] <br>
253  * For details about lambda pruning and performance comparison of SSK 
254  * and SSK-LP (SSK with lambda pruning), see [2]   
255  * Note that the complexity estimation in [2] assumes no caching of
256  * intermediate results, which has been implemented in the meantime and
257  * greatly improves the speed of the SSK without lambda pruning.
258  *<br>

376

378     setCacheSize(cacheSize);
379     setInternalCacheSize(200003);
380     setSubsequenceLength(subsequenceLength);
381     setMaxSubsequenceLength(-1);
382     setLambda(lambda);
383     
384     buildKernel(data);
385   }

519    *  The pruning method to use:
520    *  0 = No pruning
521    *  1 = Lambda pruning
522    *  (default: 0)</pre>
523    * 
524    * <pre> -C &lt;num&gt;
525    *  The size of the cache (a prime number).
526    *  (default: 250007)</pre>
527    * 
528    * <pre> -IC &lt;num&gt;
529    *  The size of the internal cache (a prime number).
530    *  (default: 200003)</pre>
531    * 
532    * <pre> -L &lt;num&gt;
533    *  The lambda constant. Penalizes non-continuous subsequence
534    *  matches. Must be in (0,1).
535    *  (default: 0.5)</pre>
536    * 
537    * <pre> -ssl &lt;num&gt;
538    *  The length of the subsequence.
539    *  (default: 3)</pre>
540    * 
541    * <pre> -ssl-max &lt;num&gt;
542    *  The maximum length of the subsequence.
543    *  (default: 9)</pre>
544    * 
545    * <pre> -N
546    *  Use normalization.
547    *  (default: no)</pre>
548    * 
549    <!-- options-end -->
550    * 
551    * @param options     the list of options as an array of strings
552    * @throws Exception  if an option is not supported
553    */
554   public void setOptions(String[] options) throws Exception {
555     String  tmpStr;
556     
557     tmpStr = Utils.getOption('P', options);
558     if (tmpStr.length() != 0)
559       setPruningMethod(
560       new SelectedTag(Integer.parseInt(tmpStr), TAGS_PRUNING));
561     else
562       setPruningMethod(
563       new SelectedTag(PRUNING_NONE, TAGS_PRUNING));
564 
565     tmpStr = Utils.getOption('C', options);
566     if (tmpStr.length() != 0)
567       setCacheSize(Integer.parseInt(tmpStr));
568     else
569       setCacheSize(250007);
570     
571     tmpStr = Utils.getOption("IC", options);
572     if (tmpStr.length() != 0)
573       setInternalCacheSize(Integer.parseInt(tmpStr));
574     else
575       setInternalCacheSize(200003);
576     
577     tmpStr = Utils.getOption('L', options);
578     if (tmpStr.length() != 0)
579       setLambda(Double.parseDouble(tmpStr));
580     else
581       setLambda(0.5);
582     
583     tmpStr = Utils.getOption("ssl", options);
584     if (tmpStr.length() != 0)
585       setSubsequenceLength(Integer.parseInt(tmpStr));
586     else
587       setSubsequenceLength(3);
588     
589     tmpStr = Utils.getOption("ssl-max", options);
590     if (tmpStr.length() != 0)
591       setMaxSubsequenceLength(Integer.parseInt(tmpStr));
592     else
593       setMaxSubsequenceLength(9);
594 
595     setUseNormalization(Utils.getFlag('N', options));
596 
597     if (getMaxSubsequenceLength()<2*getSubsequenceLength()) {
598       throw new IllegalArgumentException("Lambda Pruning forbids even contiguous substring matches! " +
599       "Use a bigger value for ssl-max (at least 2*ssl).");
600     }
601     
602     super.setOptions(options);
603   }

611     int       i;
612     Vector    result;
613     String[]  options;
614 
615     result = new Vector();
616     options = super.getOptions();
617     for (i = 0; i < options.length; i++)
618       result.add(options[i]);
619 
620     result.add("-P");
621     result.add("" + m_PruningMethod);
622 
623     result.add("-C");
624     result.add("" + getCacheSize());
625 
626     result.add("-IC");
627     result.add("" + getInternalCacheSize());
628 
629     result.add("-L");
630     result.add("" + getLambda());
631 
632     result.add("-ssl");
633     result.add("" + getSubsequenceLength());
634 
635     result.add("-ssl-max");
636     result.add("" + getMaxSubsequenceLength());
637 
638     if (getUseNormalization())

861   /**
862    * Computes the result of the kernel function for two instances.
863    * If id1 == -1, eval use inst1 instead of an instance in the dataset.
864    *
865    * @param id1 the index of the first instance in the dataset
866    * @param id2 the index of the second instance in the dataset
867    * @param inst1 the instance corresponding to id1 (used if id1 == -1)
868    * @return the result of the kernel function
869    * @throws Exception if something goes wrong
870    */
871   public double eval(int id1, int id2, Instance inst1) throws Exception {
872     if (m_Debug && id1>-1 && id2>-1) {
873       System.err.println("\nEvaluation of string kernel for");
874       System.err.println(m_data.instance(id1).stringValue(m_strAttr));
875       System.err.println("and");
876       System.err.println(m_data.instance(id2).stringValue(m_strAttr));
877     }
878 
879     //the normalized kernel returns 1 for comparison of 
880     //two identical strings
881     if (id1 == id2 && m_normalize) 
882       return 1.0;
883 
884     double result = 0;
885     long key = -1;
886     int location = -1;
887 
888     // we can only cache if we know the indexes
889     if ((id1 >= 0) && (m_keys != null)) {
890       if (id1 > id2) {
891         key = (long)id1 * m_numInsts + id2;
892       } else {
893         key = (long)id2 * m_numInsts + id1;
894       }
895       if (key < 0) {
896         throw new Exception("Cache overflow detected!");
897       }
898       location = (int)(key % m_keys.length);
899       if (m_keys[location] == (key + 1)) {
900         if (m_Debug) 
901           System.err.println("result (cached): " + m_storage[location]);
902         return m_storage[location];
903       }
904     }
905 
906     m_kernelEvals++;
907     long start = System.currentTimeMillis();
908 
909     Instance inst2 = m_data.instance(id2);
910     char[] s1 = inst1.stringValue(m_strAttr).toCharArray();
911     char[] s2 = inst2.stringValue(m_strAttr).toCharArray();
912 
913     // prevent the kernel from returning NaN
914     if (s1.length == 0 || s2.length == 0) return 0;
915 
916     if (m_normalize) {
917       result = normalizedKernel(s1,s2);
918     } else {
919       result = unnormalizedKernel(s1, s2);
920     }
921 
922     if (m_Debug) {
923       long duration = System.currentTimeMillis() - start;
924       System.err.println("result: " + result);
925       System.err.println("evaluation time:" + duration +"\n");
926     }
927 
928     // store result in cache
929     if (key != -1){
930       m_storage[location] = result;
931       m_keys[location] = (key + 1);
932     }
933     return result;
934   }

968    * evaluates the normalized kernel between s and t. See [1] for details about
969    * the normalized SSK. 
970    *
971    * @param s first input string
972    * @param t second input string

982   /**
983    * evaluates the unnormalized kernel between s and t. See [1] for details
984    * about the unnormalized SSK.
985    *
986    * @param s first input string
987    * @param t second input string
988    * @return a double indicating their distance, or similarity
989    */
990   public double unnormalizedKernel(char[] s, char[] t){
991     if (t.length > s.length) {
992       //swap because the algorithm is faster if s is
993       //the longer string
994       char[] buf = s;
995       s = t;
996       t = buf;
997     }
998     if (m_PruningMethod == PRUNING_NONE) {
999       m_multX=(s.length+1)*(t.length+1); 
1000      m_multY=(t.length+1); 
1001      m_multZ=1;
1002      maxCache = m_internalCacheSize;
1003      if (maxCache==0) {
1004    maxCache=(m_subsequenceLength+1)*m_multX;
1005      }
1006      else if ((m_subsequenceLength+1)*m_multX<maxCache) { 
1007    maxCache=(m_subsequenceLength+1)*m_multX; 
1008      }
1009      m_useRecursionCache=true;
1010      cachekhK = new int[maxCache];
1011      cachekh2K = new int[maxCache];
1012      cachekh = new double[maxCache];
1013      cachekh2 = new double[maxCache];
1014    } else if (m_PruningMethod == PRUNING_LAMBDA) {
1015      maxCache=0; 
1016      m_useRecursionCache=false;
1017    }
1018
1019    double res;
1020    if (m_PruningMethod == PRUNING_LAMBDA) {
1021      res = kernelLP(
1022              m_subsequenceLength,s,s.length-1,t,t.length-1,
1023              m_maxSubsequenceLength);
1024    } else {
1025      res = kernel(
1026              m_subsequenceLength,s,s.length-1, t, t.length-1);
1027    }
1028    cachekh = null;
1029    cachekhK = null;
1030    cachekh2 = null;
1031    cachekh2K = null;
1032    
1033    return res;
1034  }

1047  /**
1048   * the kernel function (Kn). This function performs the outer loop
1049   * character-wise over the first input string s. For each character
1050   * encountered, a recursion branch is started that identifies all
1051   * subsequences in t starting with that character. <br> See [1] for details
1052   * but note that this code is optimized and may be hard to recognize.
1053   * 
1054   * @param n the current length of the matching subsequence
1055   * @param s first string, as a char array
1056   * @param t second string, as a char array
1057   * @param endIndexS the portion of s currently regarded is s[1:endIndexS]
1058   * @param endIndexT the portion of t currently regarded is t[1:endIndexT]
1059   * @return a double indicating the distance or similarity between s and t, 
1060   * according to and depending on the initial value for n.
1061   */
1062  protected double kernel(int n, char[] s,int endIndexS, char[] t, 
1063    int endIndexT) {
1064
1065    //normal recursion ending case
1066    if (Math.min(endIndexS+1,endIndexT+1) < n) return getReturnValue(n);
1067
1068    //accumulate all recursion results in one:
1069    double result = 0;
1070
1071    //the tail-recursive function defined in [1] is turned into a
1072    //loop here, preventing stack overflows.
1073    //skim s from back to front
1074    for (int iS=endIndexS; iS > n-2; iS--) {
1075      double buf = 0;
1076      //let the current character in s be x 
1077      char x = s[iS];
1078      // iterate over all occurrences of x in t
1079      for (int j=0; j <= endIndexT; j++) {
1080        if (t[j] == x){
1081          //this is a match for the current character, hence
1082          //1. use previous chars in both strings (iS-1, j-1)
1083          //2. decrement the remainingMatchLength (n-1)
1084          //and start a recursion branch for these parameters
1085          buf += kernelHelper(n-1,s,iS-1, t, j-1);
1086        }
1087      }
1088      //ok, all occurrences of x in t have been found
1089      //multiply the result with lambda^2
1090      //  (one lambda for x, and the other for all matches of x in t)
1091      result += buf * m_powersOflambda[2];
1092    }
1093    return result;
1094  }

1305    }
1306
1307    //lambda pruning check
1308    //stops recursion if the match is so long that the resulting
1309    //power of lambda is smaller than minLambda
1310    //if lambda pruning is not used, the remainingMatchLength is < 0
1311    //and this check never stops the recursion
1312    if (remainingMatchLength < 2*n) { 
1313      return getReturnValue(n);
1314    }
1315    int adr=0;
1316    if (m_useRecursionCache) {
1317      adr = m_multX*n+m_multY*endIndexS+m_multZ*endIndexT 
1318            + m_multZZ * remainingMatchLength;
1319      if (cachekh2K[adr % maxCache]==adr+1) { 
1320        return cachekh2[adr % maxCache]; 
1321      }
1322    }
1323
1324    int rml = 0; //counts the remaining match length
1325    double result = 0;
1326    //see code docs in kernelHelper()
1327    //difference to implementation in kernelHelper:
1328    //*)choose different starting point, which is found counting
1329    //the maximal remaining match length from endIndexS.
1330    //*)keep track of the remaining match length, rml, which is
1331    //  incremented each loop
1332    for (int iS = (endIndexS-remainingMatchLength); iS <= endIndexS;iS++) {
1333      result *= m_lambda;
1334      result += kernelHelper2LP(n,s,iS, t, endIndexT,rml++);
1335    }
1336
1337    if (m_useRecursionCache && endIndexS >= 0 && endIndexT >= 0 && n >= 0) {
1338      cachekhK[adr % maxCache]=adr+1; cachekh[adr % maxCache]=result; 
1339    }
1340    return result;
1341  }

1343  /**
1344   * helper function for the evaluation of the kernel (K''n) using lambda
1345   * pruning
1346   *
1347   * @param n the current length of the matching subsequence
1348   * @param s first string, as a char array
1349   * @param t second string, as a char array
1350   * @param endIndexS the portion of s currently regarded is s[1:endIndexS]
1351   * @param endIndexT the portion of t currently regarded is t[1:endIndexT]
1352   * @param remainingMatchLength the number of characters that may still be
1353   * used 
1354   * for matching (i.e. gaps + matches in both strings)
1355   * @return a partial result for K' 
1356   */
1357  protected double kernelHelper2LP(int n, char[] s, int endIndexS,char[] t,
1358    int endIndexT,int remainingMatchLength) {
1359
1360    //lambda pruning check
1361    //stops recursion if the match is so long that the resulting
1362    //power of lambda is smaller than minLambda
1363    //if lambda pruning is not used, the remainingMatchLength is < 0
1364    //and this check never stops the recursion
1365    //if (remainingMatchLength <= 0) return 0;
1366    if (remainingMatchLength < 2*n) return getReturnValue(n);
1367
1368    //see code docs in kernelHelper2()
1369    if (endIndexS <0 || endIndexT <0) return getReturnValue(n);
1370    int adr=0;
1371    if (m_useRecursionCache){
1372      adr = m_multX*n+m_multY*endIndexS+m_multZ*endIndexT
1373            + m_multZZ * remainingMatchLength;
1374      if (cachekh2K[adr % maxCache]==adr+1) { 
1375        return cachekh2[adr % maxCache]; 
1376      }
1377    }
1378
1379    char x = s[endIndexS];
1380    if (x == t[endIndexT]) {
1381      double ret = 
1382        m_lambda 
1383        * (kernelHelper2LP(n,s,endIndexS,t,endIndexT-1,remainingMatchLength-1)
1384        + m_lambda 
1385        * kernelHelperLP(n-1,s,endIndexS-1,t,endIndexT-1,remainingMatchLength-2));
1386      if (m_useRecursionCache && endIndexS >= 0 && endIndexT >= 0 && n >= 0) {
1387        cachekh2K[adr % maxCache]=adr+1; cachekh2[adr % maxCache]=ret; }
1388      return ret;
1389    }
1390
1391    //see code docs in kernelHelper()
1392    //differences to implementation in kernelHelper():
1393    //*) choose a different ending point for the loop
1394    //   based on the remaining match length
1395    int i;
1396    int minIndex = endIndexT - remainingMatchLength;
1397    if (minIndex < 0) minIndex = 0;
1398    for (i=endIndexT; i >= minIndex;i--) {
1399      if (x == t[i]) {
1400        int skipLength = endIndexT -i;
1401        double ret = getPowerOfLambda(skipLength) *
1402          kernelHelper2LP(n,s,endIndexS,t,i,remainingMatchLength-skipLength);
1403        if (m_useRecursionCache && endIndexS >= 0 && endIndexT >= 0 && n >= 0) { 
1404          cachekh2K[adr % maxCache]=adr+1; cachekh2[adr % maxCache]=ret; 
1405        }
1406        return ret;
1407      }
1408    }
1409    double ret = getReturnValue(n);
1410    if (m_useRecursionCache && endIndexS >= 0 && endIndexT >= 0 && n >= 0) { 
1411      cachekh2K[adr % maxCache]=adr+1; cachekh2[adr % maxCache]=ret; 
1412    }
1413    return ret;
1414  }

1416  /**
1417   * precalculates small powers of lambda to speed up the kernel evaluation
1418   *
1419   * @return        the powers
1420   */
1421  private double[] calculatePowersOfLambda(){
1422    double[] powers = new double[MAX_POWER_OF_LAMBDA+1];
1423    powers[0] = 1.0;
1424    double val = 1.0;
1425    for (int i = 1; i<=MAX_POWER_OF_LAMBDA;i++) {
1426      val *= m_lambda;
1427      powers[i] = val;
1428    }
1429    return powers;
1430  }

1432  /**
1433   * retrieves a power of lambda from the lambda cache or calculates it
1434   * directly
1435   *
1436   * @param exponent    the exponent to calculate
1437   * @return        the exponent-th power of lambda
1438   */
1439  private double getPowerOfLambda(int exponent){
1440    if (exponent > MAX_POWER_OF_LAMBDA) 
1441      return Math.pow(m_lambda,exponent);
1442    
1443    if (exponent < 0) 
1444      throw new IllegalArgumentException(
1445          "only positive powers of lambda may be computed");
1446
1447    return m_powersOflambda[exponent];
1448  }
