32  /** 
33    A Token is an occurrence of a term from the text of a field.  It consists of
34    a term's text, the start and end offset of the term in the text of the field,
35    and a type string.
36    <p>
37    The start and end offsets permit applications to re-associate a token with
38    its source text, e.g., to display highlighted query terms in a document
39    browser, or to show matching text fragments in a <abbr title="KeyWord In Context">KWIC</abbr>
40    display, etc.
41    <p>
42    The type is a string, assigned by a lexical analyzer
43    (a.k.a. tokenizer), naming the lexical or syntactic class that the token
44    belongs to.  For example an end of sentence marker token might be implemented
45    with type "eos".  The default token type is "word".  
46    <p>
47    A Token can optionally have metadata (a.k.a. Payload) in the form of a variable
48    length byte array. Use {@link TermPositions#getPayloadLength()} and 
49    {@link TermPositions#getPayload(byte[], int)} to retrieve the payloads from the index.
50    
51    <br><br>
52    
53    <p><b>NOTE:</b> As of 2.9, Token implements all {@link Attribute} interfaces
54    that are part of core Lucene and can be found in the {@code tokenattributes} subpackage.
55    Even though it is not necessary to use Token anymore, with the new TokenStream API it can
56    be used as convenience class that implements all {@link Attribute}s, which is especially useful
57    to easily switch from the old to the new TokenStream API.
58    
59    <br><br>
60  
61    <p><b>NOTE:</b> As of 2.3, Token stores the term text
62    internally as a malleable char[] termBuffer instead of
63    String termText.  The indexing code and core tokenizers
64    have been changed to re-use a single Token instance, changing
65    its buffer and other fields in-place as the Token is
66    processed.  This provides substantially better indexing
67    performance as it saves the GC cost of new'ing a Token and
68    String for every term.  The APIs that accept String
69    termText are still available but a warning about the
70    associated performance cost has been added (below).  The
71    {@link #termText()} method has been deprecated.</p>
72    
73    <p>Tokenizers and TokenFilters should try to re-use a Token
74    instance when possible for best performance, by
75    implementing the {@link TokenStream#incrementToken()} API.
76    Failing that, to create a new Token you should first use
77    one of the constructors that starts with null text.  To load
78    the token from a char[] use {@link #setTermBuffer(char[], int, int)}.
79    To load from a String use {@link #setTermBuffer(String)} or {@link #setTermBuffer(String, int, int)}.
80    Alternatively you can get the Token's termBuffer by calling either {@link #termBuffer()},
81    if you know that your text is shorter than the capacity of the termBuffer
82    or {@link #resizeTermBuffer(int)}, if there is any possibility
83    that you may need to grow the buffer. Fill in the characters of your term into this
84    buffer, with {@link String#getChars(int, int, char[], int)} if loading from a string,
85    or with {@link System#arraycopy(Object, int, Object, int, int)}, and finally call {@link #setTermLength(int)} to
86    set the length of the term text.  See <a target="_top"
87    href="https://issues.apache.org/jira/browse/LUCENE-969">LUCENE-969</a>
88    for details.</p>
89    <p>Typical Token reuse patterns:
90    <ul>
91    <li> Copying text from a string (type is reset to {@link #DEFAULT_TYPE} if not specified):<br/>
92    <pre>
93      return reusableToken.reinit(string, startOffset, endOffset[, type]);
94    </pre>
95    </li>
96    <li> Copying some text from a string (type is reset to {@link #DEFAULT_TYPE} if not specified):<br/>
97    <pre>
98      return reusableToken.reinit(string, 0, string.length(), startOffset, endOffset[, type]);
99    </pre>
100   </li>
101   </li>
102   <li> Copying text from char[] buffer (type is reset to {@link #DEFAULT_TYPE} if not specified):<br/>
103   <pre>
104     return reusableToken.reinit(buffer, 0, buffer.length, startOffset, endOffset[, type]);
105   </pre>
106   </li>
107   <li> Copying some text from a char[] buffer (type is reset to {@link #DEFAULT_TYPE} if not specified):<br/>
108   <pre>
109     return reusableToken.reinit(buffer, start, end - start, startOffset, endOffset[, type]);
110   </pre>
111   </li>
112   <li> Copying from one one Token to another (type is reset to {@link #DEFAULT_TYPE} if not specified):<br/>
113   <pre>
114     return reusableToken.reinit(source.termBuffer(), 0, source.termLength(), source.startOffset(), source.endOffset()[, source.type()]);
115   </pre>
116   </li>
117   </ul>
118   A few things to note:
119   <ul>
120   <li>clear() initializes all of the fields to default values. This was changed in contrast to Lucene 2.4, but should affect no one.</li>
121   <li>Because <code>TokenStreams</code> can be chained, one cannot assume that the <code>Token's</code> current type is correct.</li>
122   <li>The startOffset and endOffset represent the start and offset in the source text, so be careful in adjusting them.</li>
123   <li>When caching a reusable token, clone it. When injecting a cached token into a stream that can be reset, clone it again.</li>
124   </ul>
125   </p>

294   /** Set the position increment.  This determines the position of this token
295    * relative to the previous Token in a {@link TokenStream}, used in phrase
296    * searching.
297    *
298    * <p>The default value is one.
299    *
300    * <p>Some common uses for this are:<ul>
301    *
302    * <li>Set it to zero to put multiple terms in the same position.  This is
303    * useful if, e.g., a word has multiple stems.  Searches for phrases
304    * including either stem will match.  In this case, all but the first stem's
305    * increment should be set to zero: the increment of the first instance
306    * should be one.  Repeating a token with an increment of zero can also be
307    * used to boost the scores of matches on that token.
308    *
309    * <li>Set it to values greater than one to inhibit exact phrase matches.
310    * If, for example, one does not want phrases to match across removed stop
311    * words, then one could build a stop word filter that removes stop words and
312    * also sets the increment to the number of stop words removed before each
313    * non-stop word.  Then exact phrase queries will only match when the terms
314    * occur with no intervening stop words.
315    *
316    * </ul>
317    * @param positionIncrement the distance from the prior term
318    * @see org.apache.lucene.index.TermPositions
319    */
320   public void setPositionIncrement(int positionIncrement) {
321     if (positionIncrement < 0)
322       throw new IllegalArgumentException
323         ("Increment must be zero or greater: " + positionIncrement);
324     this.positionIncrement = positionIncrement;
325   }

427   /** Grows the termBuffer to at least size newSize, preserving the
428    *  existing content. Note: If the next operation is to change
429    *  the contents of the term buffer use
430    *  {@link #setTermBuffer(char[], int, int)},
431    *  {@link #setTermBuffer(String)}, or
432    *  {@link #setTermBuffer(String, int, int)}
433    *  to optimally combine the resize with the setting of the termBuffer.
434    *  @param newSize minimum size of the new termBuffer
435    *  @return newly created termBuffer with length >= newSize
436    */
437   public char[] resizeTermBuffer(int newSize) {
438     if (termBuffer == null) {
439       // The buffer is always at least MIN_BUFFER_SIZE
440       newSize = newSize < MIN_BUFFER_SIZE ? MIN_BUFFER_SIZE : newSize;
441       //Preserve termText 
442       if (termText != null) {
443         final int ttLen = termText.length();
444         newSize = newSize < ttLen ? ttLen : newSize;
445         termBuffer = new char[ArrayUtil.getNextSize(newSize)];
446         termText.getChars(0, termText.length(), termBuffer, 0);
447         termText = null;
448       } else { // no term Text, the first allocation
449         termBuffer = new char[ArrayUtil.getNextSize(newSize)];
450       }    
451     } else {
452       if(termBuffer.length < newSize){
453         // Not big enough; create a new array with slight
454         // over allocation and preserve content
455         final char[] newCharBuffer = new char[ArrayUtil.getNextSize(newSize)];
456         System.arraycopy(termBuffer, 0, newCharBuffer, 0, termBuffer.length);
457         termBuffer = newCharBuffer;
458       }
459     } 
460     return termBuffer;   
461   }
462 
463   /** Allocates a buffer char[] of at least newSize, without preserving the existing content.
464    * its always used in places that set the content 
465    *  @param newSize minimum size of the buffer
466    */
467   private void growTermBuffer(int newSize) {
468     if (termBuffer == null) {
469       // The buffer is always at least MIN_BUFFER_SIZE    
470       termBuffer = new char[ArrayUtil.getNextSize(newSize < MIN_BUFFER_SIZE ? MIN_BUFFER_SIZE : newSize)];   
471     } else {
472       if(termBuffer.length < newSize){
473         // Not big enough; create a new array with slight
474         // over allocation:
475         termBuffer = new char[ArrayUtil.getNextSize(newSize)];
476       }
477     } 
478   }

481   // TODO: once we remove the deprecated termText() method
482   // and switch entirely to char[] termBuffer we don't need
483   // to use this method anymore, only for late init of the buffer
484   private void initTermBuffer() {
485     if (termBuffer == null) {
486       if (termText == null) {
487         termBuffer = new char[ArrayUtil.getNextSize(MIN_BUFFER_SIZE)];
488         termLength = 0;
489       } else {
490         int length = termText.length();
491         if (length < MIN_BUFFER_SIZE) length = MIN_BUFFER_SIZE;
492         termBuffer = new char[ArrayUtil.getNextSize(length)];
493         termLength = termText.length();
494         termText.getChars(0, termText.length(), termBuffer, 0);
495         termText = null;
496       }
497     } else {
498       termText = null;
499     }
500   }

605   public String toString() {
606     StringBuffer sb = new StringBuffer();
607     sb.append('(');
608     initTermBuffer();
609     if (termBuffer == null)
610       sb.append("null");
611     else
612       sb.append(termBuffer, 0, termLength);
613       sb.append(',').append(startOffset).append(',').append(endOffset);
614     if (!type.equals("word"))
615       sb.append(",type=").append(type);
616     if (positionIncrement != 1)
617       sb.append(",posIncr=").append(positionIncrement);
618     sb.append(')');
619     return sb.toString();
620   }

636   public Object clone() {
637     Token t = (Token)super.clone();
638     // Do a deep clone
639     if (termBuffer != null) {
640       t.termBuffer = (char[]) termBuffer.clone();
641     }
642     if (payload != null) {
643       t.payload = (Payload) payload.clone();
644     }
645     return t;
646   }

648   /** Makes a clone, but replaces the term buffer &
649    * start/end offset in the process.  This is more
650    * efficient than doing a full clone (and then calling
651    * setTermBuffer) because it saves a wasted copy of the old
652    * termBuffer. */
653   public Token clone(char[] newTermBuffer, int newTermOffset, int newTermLength, int newStartOffset, int newEndOffset) {
654     final Token t = new Token(newTermBuffer, newTermOffset, newTermLength, newStartOffset, newEndOffset);
655     t.positionIncrement = positionIncrement;
656     t.flags = flags;
657     t.type = type;
658     if (payload != null)
659       t.payload = (Payload) payload.clone();
660     return t;

663   public boolean equals(Object obj) {
664     if (obj == this)
665       return true;
666 
667     if (obj instanceof Token) {
668       Token other = (Token) obj;
669 
670       initTermBuffer();
671       other.initTermBuffer();
672       
673       if (termLength == other.termLength &&
674           startOffset == other.startOffset &&
675           endOffset == other.endOffset && 
676           flags == other.flags &&
677           positionIncrement == other.positionIncrement &&
678           subEqual(type, other.type) &&
679           subEqual(payload, other.payload)) {
680         for(int i=0;i<termLength;i++)
681           if (termBuffer[i] != other.termBuffer[i])
682             return false;
683         return true;
684       } else
685         return false;
686     } else
687       return false;
688   }

690   private boolean subEqual(Object o1, Object o2) {
691     if (o1 == null)
692       return o2 == null;
693     else
694       return o1.equals(o2);

697   public int hashCode() {
698     initTermBuffer();
699     int code = termLength;
700     code = code * 31 + startOffset;
701     code = code * 31 + endOffset;
702     code = code * 31 + flags;
703     code = code * 31 + positionIncrement;
704     code = code * 31 + type.hashCode();
705     code = (payload == null ? code : code * 31 + payload.hashCode());
706     code = code * 31 + ArrayUtil.hashCode(termBuffer, 0, termLength);
707     return code;
708   }

858   public void copyTo(AttributeImpl target) {
859     if (target instanceof Token) {
860       final Token to = (Token) target;
861       to.reinit(this);
862       // reinit shares the payload, so clone it:
863       if (payload !=null) {
864         to.payload = (Payload) payload.clone();
865       }
866     // remove the following optimization in 3.0 when old TokenStream API removed:
867     } else if (target instanceof TokenWrapper) {
868       ((TokenWrapper) target).delegate = (Token) this.clone();
869     } else {
870       initTermBuffer();
871       ((TermAttribute) target).setTermBuffer(termBuffer, 0, termLength);
872       ((OffsetAttribute) target).setOffset(startOffset, endOffset);
873       ((PositionIncrementAttribute) target).setPositionIncrement(positionIncrement);
874       ((PayloadAttribute) target).setPayload((payload == null) ? null : (Payload) payload.clone());
875       ((FlagsAttribute) target).setFlags(flags);
876       ((TypeAttribute) target).setType(type);
877     }
878   }
