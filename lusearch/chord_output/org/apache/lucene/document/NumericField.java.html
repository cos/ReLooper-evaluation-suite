30  /**
31   * <p>This class provides a {@link Field} that enables indexing
32   * of numeric values for efficient range filtering and
33   * sorting.  Here's an example usage, adding an int value:
34   * <pre>
35   *  document.add(new NumericField(name).setIntValue(value));
36   * </pre>
37   *
38   * For optimal performance, re-use the
39   * <code>NumericField</code> and {@link Document} instance for more than
40   * one document:
41   *
42   * <pre>
43   *  NumericField field = new NumericField(name);
44   *  Document document = new Document();
45   *  document.add(field);
46   *
47   *  for(all documents) {
48   *    ...
49   *    field.setIntValue(value)
50   *    writer.addDocument(document);
51   *    ...
52   *  }
53   * </pre>
54   *
55   * <p>The java native types <code>int</code>, <code>long</code>,
56   * <code>float</code> and <code>double</code> are
57   * directly supported.  However, any value that can be
58   * converted into these native types can also be indexed.
59   * For example, date/time values represented by a
60   * {@link java.util.Date} can be translated into a long
61   * value using the {@link java.util.Date#getTime} method.  If you
62   * don't need millisecond precision, you can quantize the
63   * value, either by dividing the result of
64   * {@link java.util.Date#getTime} or using the separate getters
65   * (for year, month, etc.) to construct an <code>int</code> or
66   * <code>long</code> value.</p>
67   *
68   * <p>To perform range querying or filtering against a
69   * <code>NumericField</code>, use {@link NumericRangeQuery} or {@link
70   * NumericRangeFilter}.  To sort according to a
71   * <code>NumericField</code>, use the normal numeric sort types, eg
72   * {@link SortField#INT} (note that {@link SortField#AUTO}
73   * will not work with these fields).  <code>NumericField</code> values
74   * can also be loaded directly from {@link FieldCache}.</p>
75   *
76   * <p>By default, a <code>NumericField</code>'s value is not stored but
77   * is indexed for range filtering and sorting.  You can use
78   * the {@link #NumericField(String,Field.Store,boolean)}
79   * constructor if you need to change these defaults.</p>
80   *
81   * <p>You may add the same field name as a <code>NumericField</code> to
82   * the same document more than once.  Range querying and
83   * filtering will be the logical OR of all values; so a range query
84   * will hit all documents that have at least one value in
85   * the range. However sort behavior is not defined.  If you need to sort,
86   * you should separately index a single-valued <code>NumericField</code>.</p>
87   *
88   * <p>A <code>NumericField</code> will consume somewhat more disk space
89   * in the index than an ordinary single-valued field.
90   * However, for a typical index that includes substantial
91   * textual content per document, this increase will likely
92   * be in the noise. </p>
93   *
94   * <p>Within Lucene, each numeric value is indexed as a
95   * <em>trie</em> structure, where each term is logically
96   * assigned to larger and larger pre-defined brackets (which
97   * are simply lower-precision representations of the value).
98   * The step size between each successive bracket is called the
99   * <code>precisionStep</code>, measured in bits.  Smaller
100  * <code>precisionStep</code> values result in larger number
101  * of brackets, which consumes more disk space in the index
102  * but may result in faster range search performance.  The
103  * default value, 4, was selected for a reasonable tradeoff
104  * of disk space consumption versus performance.  You can
105  * use the expert constructor {@link
106  * #NumericField(String,int,Field.Store,boolean)} if you'd
107  * like to change the value.  Note that you must also
108  * specify a congruent value when creating {@link
109  * NumericRangeQuery} or {@link NumericRangeFilter}.
110  * For low cardinality fields larger precision steps are good.
111  * If the cardinality is &lt; 100, it is fair
112  * to use {@link Integer#MAX_VALUE}, which produces one
113  * term per value.
114  *
115  * <p>For more information on the internals of numeric trie
116  * indexing, including the <a
117  * href="../search/NumericRangeQuery.html#precisionStepDesc"><code>precisionStep</code></a>
118  * configuration, see {@link NumericRangeQuery}. The format of
119  * indexed values is described in {@link NumericUtils}.
120  *
121  * <p>If you only need to sort by numeric value, and never
122  * run range querying/filtering, you can index using a
123  * <code>precisionStep</code> of {@link Integer#MAX_VALUE}.
124  * This will minimize disk space consumed. </p>
125  *
126  * <p>More advanced users can instead use {@link
127  * NumericTokenStream} directly, when indexing numbers. This
128  * class is a wrapper around this token stream type for
129  * easier, more intuitive usage.</p>
130  *
131  * <p><b>NOTE:</b> This class is only used during
132  * indexing. When retrieving the stored field value from a
133  * {@link Document} instance after search, you will get a
134  * conventional {@link Fieldable} instance where the numeric
135  * values are returned as {@link String}s (according to
136  * <code>toString(value)</code> of the used data type).
137  *
138  * <p><font color="red"><b>NOTE:</b> This API is
139  * experimental and might change in incompatible ways in the
140  * next release.</font>
141  *
142  * @since 2.9
143  */
144 public final class NumericField extends AbstractField {
145 
146   private final NumericTokenStream tokenStream;
147 
148   /**
149    * Creates a field for numeric values using the default <code>precisionStep</code>
150    * {@link NumericUtils#PRECISION_STEP_DEFAULT} (4). The instance is not yet initialized with
151    * a numeric value, before indexing a document containing this field,
152    * set a value using the various set<em>???</em>Value() methods.
153    * This constructor creates an indexed, but not stored field.
154    * @param name the field name
155    */
156   public NumericField(String name) {
157     this(name, NumericUtils.PRECISION_STEP_DEFAULT, Field.Store.NO, true);
158   }
159   
160   /**
161    * Creates a field for numeric values using the default <code>precisionStep</code>
162    * {@link NumericUtils#PRECISION_STEP_DEFAULT} (4). The instance is not yet initialized with
163    * a numeric value, before indexing a document containing this field,
164    * set a value using the various set<em>???</em>Value() methods.
165    * @param name the field name
166    * @param store if the field should be stored in plain text form
167    *  (according to <code>toString(value)</code> of the used data type)
168    * @param index if the field should be indexed using {@link NumericTokenStream}
169    */
170   public NumericField(String name, Field.Store store, boolean index) {
171     this(name, NumericUtils.PRECISION_STEP_DEFAULT, store, index);
172   }
173   
174   /**
175    * Creates a field for numeric values with the specified
176    * <code>precisionStep</code>. The instance is not yet initialized with
177    * a numeric value, before indexing a document containing this field,
178    * set a value using the various set<em>???</em>Value() methods.
179    * This constructor creates an indexed, but not stored field.
180    * @param name the field name
181    * @param precisionStep the used <a href="../search/NumericRangeQuery.html#precisionStepDesc">precision step</a>
182    */
183   public NumericField(String name, int precisionStep) {
184     this(name, precisionStep, Field.Store.NO, true);
185   }
186 
187   /**
188    * Creates a field for numeric values with the specified
189    * <code>precisionStep</code>. The instance is not yet initialized with
190    * a numeric value, before indexing a document containing this field,
191    * set a value using the various set<em>???</em>Value() methods.
192    * @param name the field name
193    * @param precisionStep the used <a href="../search/NumericRangeQuery.html#precisionStepDesc">precision step</a>
194    * @param store if the field should be stored in plain text form
195    *  (according to <code>toString(value)</code> of the used data type)
196    * @param index if the field should be indexed using {@link NumericTokenStream}
197    */
198   public NumericField(String name, int precisionStep, Field.Store store, boolean index) {
199     super(name, store, index ? Field.Index.ANALYZED_NO_NORMS : Field.Index.NO, Field.TermVector.NO);
200     setOmitTermFreqAndPositions(true);
201     tokenStream = new NumericTokenStream(precisionStep);
202   }
203 
204   /** Returns a {@link NumericTokenStream} for indexing the numeric value. */
205   public TokenStream tokenStreamValue()   {
206     return isIndexed() ? tokenStream : null;
207   }
208   
209   /** Returns always <code>null</code> for numeric fields */
210   public byte[] binaryValue() {
211     return null;
212   }
213   
214   /** Returns always <code>null</code> for numeric fields */
215   public byte[] getBinaryValue(byte[] result){
216     return null;
217   }
218 
219   /** Returns always <code>null</code> for numeric fields */
220   public Reader readerValue() {
221     return null;
222   }
223     
224   /** Returns the numeric value as a string (how it is stored, when {@link Field.Store#YES} is chosen). */
225   public String stringValue()   {
226     return (fieldsData == null) ? null : fieldsData.toString();
227   }
228   
229   /** Returns the current numeric value as a subclass of {@link Number}, <code>null</code> if not yet initialized. */
230   public Number getNumericValue() {
231     return (Number) fieldsData;
232   }
233   
234   /**
235    * Initializes the field with the supplied <code>long</code> value.
236    * @param value the numeric value
237    * @return this instance, because of this you can use it the following way:
238    * <code>document.add(new NumericField(name, precisionStep).setLongValue(value))</code>
239    */
240   public NumericField setLongValue(final long value) {
241     tokenStream.setLongValue(value);
242     fieldsData = new Long(value);
243     return this;
244   }
245   
246   /**
247    * Initializes the field with the supplied <code>int</code> value.
248    * @param value the numeric value
249    * @return this instance, because of this you can use it the following way:
250    * <code>document.add(new NumericField(name, precisionStep).setIntValue(value))</code>
251    */
252   public NumericField setIntValue(final int value) {
253     tokenStream.setIntValue(value);
254     fieldsData = new Integer(value);
255     return this;
256   }
257   
258   /**
259    * Initializes the field with the supplied <code>double</code> value.
260    * @param value the numeric value
261    * @return this instance, because of this you can use it the following way:
262    * <code>document.add(new NumericField(name, precisionStep).setDoubleValue(value))</code>
263    */
264   public NumericField setDoubleValue(final double value) {
265     tokenStream.setDoubleValue(value);
266     fieldsData = new Double(value);
267     return this;
268   }
269   
270   /**
271    * Initializes the field with the supplied <code>float</code> value.
272    * @param value the numeric value
273    * @return this instance, because of this you can use it the following way:
274    * <code>document.add(new NumericField(name, precisionStep).setFloatValue(value))</code>
275    */
276   public NumericField setFloatValue(final float value) {
277     tokenStream.setFloatValue(value);
278     fieldsData = new Float(value);
279     return this;
280   }
281 
282 }
