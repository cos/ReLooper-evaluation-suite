46  /**
47    An <code>IndexWriter</code> creates and maintains an index.
48  
49    <p>The <code>create</code> argument to the {@link
50    #IndexWriter(Directory, Analyzer, boolean) constructor} determines 
51    whether a new index is created, or whether an existing index is
52    opened.  Note that you can open an index with <code>create=true</code>
53    even while readers are using the index.  The old readers will 
54    continue to search the "point in time" snapshot they had opened, 
55    and won't see the newly created index until they re-open.  There are
56    also {@link #IndexWriter(Directory, Analyzer) constructors}
57    with no <code>create</code> argument which will create a new index
58    if there is not already an index at the provided path and otherwise 
59    open the existing index.</p>
60  
61    <p>In either case, documents are added with {@link #addDocument(Document)
62    addDocument} and removed with {@link #deleteDocuments(Term)} or {@link
63    #deleteDocuments(Query)}. A document can be updated with {@link
64    #updateDocument(Term, Document) updateDocument} (which just deletes
65    and then adds the entire document). When finished adding, deleting 
66    and updating documents, {@link #close() close} should be called.</p>
67  
68    <a name="flush"></a>
69    <p>These changes are buffered in memory and periodically
70    flushed to the {@link Directory} (during the above method
71    calls).  A flush is triggered when there are enough
72    buffered deletes (see {@link #setMaxBufferedDeleteTerms})
73    or enough added documents since the last flush, whichever
74    is sooner.  For the added documents, flushing is triggered
75    either by RAM usage of the documents (see {@link
76    #setRAMBufferSizeMB}) or the number of added documents.
77    The default is to flush when RAM usage hits 16 MB.  For
78    best indexing speed you should flush by RAM usage with a
79    large RAM buffer.  Note that flushing just moves the
80    internal buffered state in IndexWriter into the index, but
81    these changes are not visible to IndexReader until either
82    {@link #commit()} or {@link #close} is called.  A flush may
83    also trigger one or more segment merges which by default
84    run with a background thread so as not to block the
85    addDocument calls (see <a href="#mergePolicy">below</a>
86    for changing the {@link MergeScheduler}).</p>
87  
88    <a name="autoCommit"></a>
89    <p>The optional <code>autoCommit</code> argument to the {@link
90    #IndexWriter(Directory, boolean, Analyzer) constructors}
91    controls visibility of the changes to {@link IndexReader}
92    instances reading the same index.  When this is
93    <code>false</code>, changes are not visible until {@link
94    #close()} or {@link #commit()} is called.  Note that changes will still be
95    flushed to the {@link Directory} as new files, but are 
96    not committed (no new <code>segments_N</code> file is written 
97    referencing the new files, nor are the files sync'd to stable storage)
98    until {@link #close()} or {@link #commit()} is called.  If something
99    goes terribly wrong (for example the JVM crashes), then
100   the index will reflect none of the changes made since the
101   last commit, or the starting state if commit was not called.
102   You can also call {@link #rollback()}, which closes the writer
103   without committing any changes, and removes any index
104   files that had been flushed but are now unreferenced.
105   This mode is useful for preventing readers from refreshing
106   at a bad time (for example after you've done all your
107   deletes but before you've done your adds).  It can also be
108   used to implement simple single-writer transactional
109   semantics ("all or none").  You can do a two-phase commit
110   by calling {@link #prepareCommit()}
111   followed by {@link #commit()}. This is necessary when
112   Lucene is working with an external resource (for example,
113   a database) and both must either commit or rollback the
114   transaction.</p>
115 
116   <p>When <code>autoCommit</code> is <code>true</code> then
117   the writer will periodically commit on its own.  [<b>Deprecated</b>: Note that in 3.0, IndexWriter will
118   no longer accept autoCommit=true (it will be hardwired to
119   false).  You can always call {@link #commit()} yourself
120   when needed]. There is
121   no guarantee when exactly an auto commit will occur (it
122   used to be after every flush, but it is now after every
123   completed merge, as of 2.4).  If you want to force a
124   commit, call {@link #commit()}, or, close the writer.  Once
125   a commit has finished, newly opened {@link IndexReader} instances will
126   see the changes to the index as of that commit.  When
127   running in this mode, be careful not to refresh your
128   readers while optimize or segment merges are taking place
129   as this can tie up substantial disk space.</p>
130   
131   <p>Regardless of <code>autoCommit</code>, an {@link
132   IndexReader} or {@link org.apache.lucene.search.IndexSearcher} will only see the
133   index as of the "point in time" that it was opened.  Any
134   changes committed to the index after the reader was opened
135   are not visible until the reader is re-opened.</p>
136 
137   <p>If an index will not have more documents added for a while and optimal search
138   performance is desired, then either the full {@link #optimize() optimize}
139   method or partial {@link #optimize(int)} method should be
140   called before the index is closed.</p>
141 
142   <p>Opening an <code>IndexWriter</code> creates a lock file for the directory in use. Trying to open
143   another <code>IndexWriter</code> on the same directory will lead to a
144   {@link LockObtainFailedException}. The {@link LockObtainFailedException}
145   is also thrown if an IndexReader on the same directory is used to delete documents
146   from the index.</p>
147   
148   <a name="deletionPolicy"></a>
149   <p>Expert: <code>IndexWriter</code> allows an optional
150   {@link IndexDeletionPolicy} implementation to be
151   specified.  You can use this to control when prior commits
152   are deleted from the index.  The default policy is {@link
153   KeepOnlyLastCommitDeletionPolicy} which removes all prior
154   commits as soon as a new commit is done (this matches
155   behavior before 2.2).  Creating your own policy can allow
156   you to explicitly keep previous "point in time" commits
157   alive in the index for some time, to allow readers to
158   refresh to the new commit without having the old commit
159   deleted out from under them.  This is necessary on
160   filesystems like NFS that do not support "delete on last
161   close" semantics, which Lucene's "point in time" search
162   normally relies on. </p>
163 
164   <a name="mergePolicy"></a> <p>Expert:
165   <code>IndexWriter</code> allows you to separately change
166   the {@link MergePolicy} and the {@link MergeScheduler}.
167   The {@link MergePolicy} is invoked whenever there are
168   changes to the segments in the index.  Its role is to
169   select which merges to do, if any, and return a {@link
170   MergePolicy.MergeSpecification} describing the merges.  It
171   also selects merges to do for optimize().  (The default is
172   {@link LogByteSizeMergePolicy}.  Then, the {@link
173   MergeScheduler} is invoked with the requested merges and
174   it decides when and how to run the merges.  The default is
175   {@link ConcurrentMergeScheduler}. </p>
176 
177   <a name="OOME"></a><p><b>NOTE</b>: if you hit an
178   OutOfMemoryError then IndexWriter will quietly record this
179   fact and block all future segment commits.  This is a
180   defensive measure in case any internal state (buffered
181   documents and deletions) were corrupted.  Any subsequent
182   calls to {@link #commit()} will throw an
183   IllegalStateException.  The only course of action is to
184   call {@link #close()}, which internally will call {@link
185   #rollback()}, to undo any changes to the index since the
186   last commit.  If you opened the writer with autoCommit
187   false you can also just call {@link #rollback()}
188   directly.</p>
189 
190   <a name="thread-safety"></a><p><b>NOTE</b>: {@link
191   <code>IndexWriter</code>} instances are completely thread
192   safe, meaning multiple threads can call any of its
193   methods, concurrently.  If your application requires
194   external synchronization, you should <b>not</b>
195   synchronize on the <code>IndexWriter</code> instance as
196   this may cause deadlock; use your own (non-Lucene) objects
197   instead. </p>
198 */

200 /*
201  * Clarification: Check Points (and commits)
202  * Being able to set autoCommit=false allows IndexWriter to flush and 
203  * write new index files to the directory without writing a new segments_N
204  * file which references these new files. It also means that the state of 
205  * the in memory SegmentInfos object is different than the most recent
206  * segments_N file written to the directory.
207  * 
208  * Each time the SegmentInfos is changed, and matches the (possibly 
209  * modified) directory files, we have a new "check point". 
210  * If the modified/new SegmentInfos is written to disk - as a new 
211  * (generation of) segments_N file - this check point is also an 
212  * IndexCommit.
213  * 
214  * With autoCommit=true, every checkPoint is also a CommitPoint.
215  * With autoCommit=false, some checkPoints may not be commits.
216  * 
217  * A new checkpoint always replaces the previous checkpoint and 
218  * becomes the new "front" of the index. This allows the IndexFileDeleter 
219  * to delete files that are referenced only by stale checkpoints.
220  * (files that were created since the last commit, but are no longer
221  * referenced by the "front" of the index). For this, IndexFileDeleter 
222  * keeps track of the last non commit checkpoint.
223  */

387   /**
388    * Expert: returns a readonly reader, covering all
389    * committed as well as un-committed changes to the index.
390    * This provides "near real-time" searching, in that
391    * changes made during an IndexWriter session can be
392    * quickly made available for searching without closing
393    * the writer nor calling {@link #commit}.
394    *
395    * <p>Note that this is functionally equivalent to calling
396    * {#commit} and then using {@link IndexReader#open} to
397    * open a new reader.  But the turarnound time of this
398    * method should be faster since it avoids the potentially
399    * costly {@link #commit}.</p>
400    *
401    * <p>You must close the {@link IndexReader} returned by
402    * this method once you are done using it.</p>
403    *
404    * <p>It's <i>near</i> real-time because there is no hard
405    * guarantee on how quickly you can get a new reader after
406    * making changes with IndexWriter.  You'll have to
407    * experiment in your situation to determine if it's
408    * fast enough.  As this is a new and experimental
409    * feature, please report back on your findings so we can
410    * learn, improve and iterate.</p>
411    *
412    * <p>The resulting reader supports {@link
413    * IndexReader#reopen}, but that call will simply forward
414    * back to this method (though this may change in the
415    * future).</p>
416    *
417    * <p>The very first time this method is called, this
418    * writer instance will make every effort to pool the
419    * readers that it opens for doing merges, applying
420    * deletes, etc.  This means additional resources (RAM,
421    * file descriptors, CPU time) will be consumed.</p>
422    *
423    * <p>For lower latency on reopening a reader, you should
424    * call {@link #setMergedSegmentWarmer} to
425    * pre-warm a newly merged segment before it's committed
426    * to the index.  This is important for minimizing
427    * index-to-search delay after a large merge.  </p>
428    *
429    * <p>If an addIndexes* call is running in another thread,
430    * then this reader will only search those segments from
431    * the foreign index that have been successfully copied
432    * over, so far</p>.
433    *
434    * <p><b>NOTE</b>: Once the writer is closed, any
435    * outstanding readers may continue to be used.  However,
436    * if you attempt to reopen any of those readers, you'll
437    * hit an {@link AlreadyClosedException}.</p>
438    *
439    * <p><b>NOTE:</b> This API is experimental and might
440    * change in incompatible ways in the next release.</p>
441    *
442    * @return IndexReader that covers entire index plus all
443    * changes made so far by this IndexWriter instance
444    *
445    * @throws IOException
446    */

485   /** Holds shared SegmentReader instances. IndexWriter uses
486    *  SegmentReaders for 1) applying deletes, 2) doing
487    *  merges, 3) handing out a real-time reader.  This pool
488    *  reuses instances of the SegmentReaders in all these
489    *  places if it is in "near real-time mode" (getReader()
490    *  has been called on this instance). */
491 
492   class ReaderPool {
493 
494     private final Map readerMap = new HashMap();
495 
496     /** Forcefully clear changes for the specified segments,
497      *  and remove from the pool.   This is called on successful merge. */
498     synchronized void clear(SegmentInfos infos) throws IOException {
499       if (infos == null) {
500         Iterator iter = readerMap.entrySet().iterator();
501         while (iter.hasNext()) {
502           Map.Entry ent = (Map.Entry) iter.next();
503           ((SegmentReader) ent.getValue()).hasChanges = false;
504         }
505       } else {
506         final int numSegments = infos.size();
507         for(int i=0;i<numSegments;i++) {
508           final SegmentInfo info = infos.info(i);
509           if (readerMap.containsKey(info)) {
510             ((SegmentReader) readerMap.get(info)).hasChanges = false;
511           }
512         }     
513       }
514     }
515     
516     // used only by asserts
517     public synchronized boolean infoIsLive(SegmentInfo info) {
518       int idx = segmentInfos.indexOf(info);
519       assert idx != -1;
520       assert segmentInfos.get(idx) == info;
521       return true;
522     }
523 
524     public synchronized SegmentInfo mapToLive(SegmentInfo info) {
525       int idx = segmentInfos.indexOf(info);
526       if (idx != -1) {
527         info = (SegmentInfo) segmentInfos.get(idx);
528       }
529       return info;
530     }

542     /**
543      * Release the segment reader (i.e. decRef it and close if there
544      * are no more references.
545      * @param sr
546      * @throws IOException
547      */
548     public synchronized void release(SegmentReader sr, boolean drop) throws IOException {
549 
550       final boolean pooled = readerMap.containsKey(sr.getSegmentInfo());
551 
552       assert !pooled | readerMap.get(sr.getSegmentInfo()) == sr;
553 
554       // Drop caller's ref
555       sr.decRef();
556 
557       if (pooled && (drop || (!poolReaders && sr.getRefCount() == 1))) {
558 
559         // We are the last ref to this reader; since we're
560         // not pooling readers, we release it:
561         readerMap.remove(sr.getSegmentInfo());
562 
563         // TODO: java 5
564         // assert !sr.hasChanges || Thread.holdsLock(IndexWriter.this);
565 
566         // Drop our ref -- this will commit any pending
567         // changes to the dir
568         boolean success = false;
569         try {
570           sr.close();
571           success = true;
572         } finally {
573           if (!success && sr.hasChanges) {
574             // Abandon the changes & retry closing:
575             sr.hasChanges = false;
576             try {
577               sr.close();
578             } catch (Throwable ignore) {
579               // Keep throwing original exception
580             }
581           }
582         }
583       }
584     }
585     
586     /** Remove all our references to readers, and commits
587      *  any pending changes. */
588     synchronized void close() throws IOException {
589       Iterator iter = readerMap.entrySet().iterator();
590       while (iter.hasNext()) {
591         Map.Entry ent = (Map.Entry) iter.next();
592 
593         SegmentReader sr = (SegmentReader) ent.getValue();
594         if (sr.hasChanges) {
595           assert infoIsLive(sr.getSegmentInfo());
596           sr.startCommit();
597           boolean success = false;
598           try {
599             sr.doCommit(null);
600             success = true;
601           } finally {
602             if (!success) {
603               sr.rollbackCommit();
604             }
605           }
606         }
607 
608         iter.remove();
609 
610         // NOTE: it is allowed that this decRef does not
611         // actually close the SR; this can happen when a
612         // near real-time reader is kept open after the
613         // IndexWriter instance is closed
614         sr.decRef();
615       }
616     }

619      * Commit all segment reader in the pool.
620      * @throws IOException
621      */
622     synchronized void commit() throws IOException {
623       Iterator iter = readerMap.entrySet().iterator();
624       while (iter.hasNext()) {
625         Map.Entry ent = (Map.Entry) iter.next();
626 
627         SegmentReader sr = (SegmentReader) ent.getValue();
628         if (sr.hasChanges) {
629           assert infoIsLive(sr.getSegmentInfo());
630           sr.startCommit();
631           boolean success = false;
632           try {
633             sr.doCommit(null);
634             success = true;
635           } finally {
636             if (!success) {
637               sr.rollbackCommit();
638             }
639           }
640         }
641       }
642     }

669     /**
670      * Obtain a SegmentReader from the readerPool.  The reader
671      * must be returned by calling {@link #release(SegmentReader)}
672      * 
673      * @see #release(SegmentReader)
674      * @param info
675      * @param doOpenStores
676      * @param readBufferSize
677      * @param termsIndexDivisor
678      * @throws IOException
679      */
680     public synchronized SegmentReader get(SegmentInfo info, boolean doOpenStores, int readBufferSize, int termsIndexDivisor) throws IOException {
681 
682       if (poolReaders) {
683         readBufferSize = BufferedIndexInput.BUFFER_SIZE;
684       }
685 
686       SegmentReader sr = (SegmentReader) readerMap.get(info);
687       if (sr == null) {
688         // TODO: we may want to avoid doing this while
689         // synchronized
690         // Returns a ref, which we xfer to readerMap:
691         sr = SegmentReader.get(info, readBufferSize, doOpenStores, termsIndexDivisor);
692         readerMap.put(info, sr);
693       } else {
694         if (doOpenStores) {
695           sr.openDocStores();
696         }
697         if (termsIndexDivisor != -1 && !sr.termsIndexLoaded()) {
698           // If this reader was originally opened because we
699           // needed to merge it, we didn't load the terms
700           // index.  But now, if the caller wants the terms
701           // index (eg because it's doing deletes, or an NRT
702           // reader is being opened) we ask the reader to
703           // load its terms index.
704           sr.loadTermsIndex(termsIndexDivisor);
705         }
706       }
707 
708       // Return a ref to our caller
709       sr.incRef();
710       return sr;
711     }

743   synchronized void acquireWrite() {
744     assert writeThread != Thread.currentThread();
745     while(writeThread != null || readCount > 0)
746       doWait();
747 
748     // We could have been closed while we were waiting:
749     ensureOpen();
750 
751     writeThread = Thread.currentThread();
752   }

755     assert Thread.currentThread() == writeThread;
756     writeThread = null;
757     notifyAll();

760   synchronized void acquireRead() {
761     final Thread current = Thread.currentThread();
762     while(writeThread != null && writeThread != current)
763       doWait();
764 
765     readCount++;

768   // Allows one readLock to upgrade to a writeLock even if
769   // there are other readLocks as long as all other
770   // readLocks are also blocked in this method:
771   synchronized void upgradeReadToWrite() {
772     assert readCount > 0;
773     upgradeCount++;
774     while(readCount > upgradeCount || writeThread != null) {
775       doWait();
776     }
777     
778     writeThread = Thread.currentThread();
779     readCount--;
780     upgradeCount--;
781   }

785     assert readCount >= 0;

1531  private void init(Directory d, Analyzer a, final boolean create, boolean closeDir, 
1532                    IndexDeletionPolicy deletionPolicy, boolean autoCommit, int maxFieldLength,
1533                    IndexingChain indexingChain, IndexCommit commit)
1534    throws CorruptIndexException, LockObtainFailedException, IOException {
1535    this.closeDir = closeDir;
1536    directory = d;
1537    analyzer = a;
1538    setMessageID(defaultInfoStream);
1539    this.maxFieldLength = maxFieldLength;
1540
1541    if (indexingChain == null)
1542      indexingChain = DocumentsWriter.DefaultIndexingChain;
1543
1544    if (create) {
1545      // Clear the write lock in case it's leftover:
1546      directory.clearLock(WRITE_LOCK_NAME);
1547    }
1548
1549    Lock writeLock = directory.makeLock(WRITE_LOCK_NAME);
1550    if (!writeLock.obtain(writeLockTimeout)) // obtain write lock
1551      throw new LockObtainFailedException("Index locked for write: " + writeLock);
1552    this.writeLock = writeLock;                   // save it
1553
1554    try {
1555      if (create) {
1556        // Try to read first.  This is to allow create
1557        // against an index that's currently open for
1558        // searching.  In this case we write the next
1559        // segments_N file with no segments:
1560        boolean doCommit;
1561        try {
1562          segmentInfos.read(directory);
1563          segmentInfos.clear();
1564          doCommit = false;
1565        } catch (IOException e) {
1566          // Likely this means it's a fresh directory
1567          doCommit = true;
1568        }
1569
1570        if (autoCommit || doCommit) {
1571          // Always commit if autoCommit=true, else only
1572          // commit if there is no segments file in this dir
1573          // already.
1574          segmentInfos.commit(directory);
1575          synced.addAll(segmentInfos.files(directory, true));
1576        } else {
1577          // Record that we have a change (zero out all
1578          // segments) pending:
1579          changeCount++;
1580        }
1581      } else {
1582        segmentInfos.read(directory);
1583
1584        if (commit != null) {
1585          // Swap out all segments, but, keep metadata in
1586          // SegmentInfos, like version & generation, to
1587          // preserve write-once.  This is important if
1588          // readers are open against the future commit
1589          // points.
1590          if (commit.getDirectory() != directory)
1591            throw new IllegalArgumentException("IndexCommit's directory doesn't match my directory");
1592          SegmentInfos oldInfos = new SegmentInfos();
1593          oldInfos.read(directory, commit.getSegmentsFileName());
1594          segmentInfos.replace(oldInfos);
1595          changeCount++;
1596          if (infoStream != null)
1597            message("init: loaded commit \"" + commit.getSegmentsFileName() + "\"");
1598        }
1599
1600        // We assume that this segments_N was previously
1601        // properly sync'd:
1602        synced.addAll(segmentInfos.files(directory, true));
1603      }
1604
1605      this.autoCommit = autoCommit;
1606      setRollbackSegmentInfos(segmentInfos);
1607
1608      docWriter = new DocumentsWriter(directory, this, indexingChain);
1609      docWriter.setInfoStream(infoStream);
1610      docWriter.setMaxFieldLength(maxFieldLength);
1611
1612      // Default deleter (for backwards compatibility) is
1613      // KeepOnlyLastCommitDeleter:
1614      deleter = new IndexFileDeleter(directory,
1615                                     deletionPolicy == null ? new KeepOnlyLastCommitDeletionPolicy() : deletionPolicy,
1616                                     segmentInfos, infoStream, docWriter);
1617
1618      if (deleter.startingCommitDeleted)
1619        // Deletion policy deleted the "head" commit point.
1620        // We have to mark ourself as changed so that if we
1621        // are closed w/o any further changes we write a new
1622        // segments_N file.
1623        changeCount++;
1624
1625      pushMaxBufferedDocs();
1626
1627      if (infoStream != null) {
1628        message("init: create=" + create);
1629        messageState();
1630      }
1631
1632    } catch (IOException e) {
1633      this.writeLock.release();
1634      this.writeLock = null;
1635      throw e;
1636    }
1637  }

1644    for(int i=0;i<size;i++)

2126   * Closes the index with or without waiting for currently
2127   * running merges to finish.  This is only meaningful when
2128   * using a MergeScheduler that runs merges in background
2129   * threads.
2130   *

2132   * you should immediately close the writer, again.  See <a
2133   * href="#OOME">above</a> for details.</p>
2134   *
2135   * <p><b>NOTE</b>: it is dangerous to always call
2136   * close(false), especially when IndexWriter is not open
2137   * for very long, because this can result in "merge
2138   * starvation" whereby long merges will never have a
2139   * chance to finish.  This will cause too many segments in
2140   * your index over time.</p>
2141   *

2143   * until all merges complete; else, it will ask all
2144   * running merges to abort, wait until those merges have
2145   * finished (which should be at most a few seconds), and
2146   * then return.
2147   */
2148  public void close(boolean waitForMerges) throws CorruptIndexException, IOException {
2149
2150    // Ensure that only one thread actually gets to do the closing:
2151    if (shouldClose()) {

2153      // on close, in case the internal state of IndexWriter

2158        closeInternal(waitForMerges);
2159    }

2162  // Returns true if this thread should attempt to close, or
2163  // false if IndexWriter is now closed; else, waits until
2164  // another thread finishes closing
2165  synchronized private boolean shouldClose() {
2166    while(true) {
2167      if (!closed) {
2168        if (!closing) {
2169          closing = true;
2170          return true;
2171        } else {
2172          // Another thread is presently trying to close;
2173          // wait until it finishes one way (closes
2174          // successfully) or another (fails to close)
2175          doWait();
2176        }
2177      } else
2178        return false;
2179    }
2180  }

2200        // any pending merges are waiting:

2707   * Requests an "optimize" operation on an index, priming the index
2708   * for the fastest available search. Traditionally this has meant
2709   * merging all segments into a single segment as is done in the
2710   * default merge policy, but individual merge policies may implement
2711   * optimize in different ways.
2712   *
2713   * <p>It is recommended that this method be called upon completion of indexing.  In
2714   * environments with frequent updates, optimize is best done during low volume times, if at all. 
2715   * 
2716   * </p>
2717   * <p>See http://www.gossamer-threads.com/lists/lucene/java-dev/47895 for more discussion. </p>
2718   *
2719   * <p>Note that optimize requires 2X the index size free
2720   * space in your Directory.  For example, if your index
2721   * size is 10 MB then you need 20 MB free for optimize to
2722   * complete.</p>
2723   *
2724   * <p>If some but not all readers re-open while an
2725   * optimize is underway, this will cause > 2X temporary
2726   * space to be consumed as those new readers will then
2727   * hold open the partially optimized segments at that
2728   * time.  It is best not to re-open readers while optimize
2729   * is running.</p>
2730   *
2731   * <p>The actual temporary usage could be much less than
2732   * these figures (it depends on many factors).</p>
2733   *
2734   * <p>In general, once the optimize completes, the total size of the
2735   * index will be less than the size of the starting index.
2736   * It could be quite a bit smaller (if there were many
2737   * pending deletes) or just slightly smaller.</p>
2738   *
2739   * <p>If an Exception is hit during optimize(), for example
2740   * due to disk full, the index will not be corrupt and no
2741   * documents will have been lost.  However, it may have
2742   * been partially optimized (some segments were merged but
2743   * not all), and it's possible that one of the segments in
2744   * the index will be in non-compound format even when
2745   * using compound file format.  This will occur when the
2746   * Exception is hit during conversion of the segment into
2747   * compound format.</p>
2748   *
2749   * <p>This call will optimize those segments present in
2750   * the index when the call started.  If other threads are
2751   * still adding documents and flushing segments, those
2752   * newly created segments will not be optimized unless you
2753   * call optimize again.</p>
2754   *
2755   * <p><b>NOTE</b>: if this method hits an OutOfMemoryError
2756   * you should immediately close the writer.  See <a
2757   * href="#OOME">above</a> for details.</p>
2758   *
2759   * @throws CorruptIndexException if the index is corrupt
2760   * @throws IOException if there is a low-level IO error
2761   * @see LogMergePolicy#findMergesForOptimize
2762  */

2797  /** Just like {@link #optimize(int)}, except you can
2798   *  specify whether the call should block until the
2799   *  optimize completes.  This is only meaningful with a
2800   *  {@link MergeScheduler} that is able to run merges in
2801   *  background threads.
2802   *
2803   * <p><b>NOTE</b>: if this method hits an OutOfMemoryError
2804   * you should immediately close the writer.  See <a
2805   * href="#OOME">above</a> for details.</p>
2806   */
2807  public void optimize(int maxNumSegments, boolean doWait) throws CorruptIndexException, IOException {
2808    ensureOpen();
2809
2810    if (maxNumSegments < 1)
2811      throw new IllegalArgumentException("maxNumSegments must be >= 1; got " + maxNumSegments);
2812
2813    if (infoStream != null)
2814      message("optimize: index now " + segString());
2815
2816    flush(true, false, true);
2817
2818    synchronized(this) {
2819      resetMergeExceptions();
2820      segmentsToOptimize = new HashSet();
2821      final int numSegments = segmentInfos.size();
2822      for(int i=0;i<numSegments;i++)
2823        segmentsToOptimize.add(segmentInfos.info(i));
2824      
2825      // Now mark all pending & running merges as optimize
2826      // merge:
2827      Iterator it = pendingMerges.iterator();
2828      while(it.hasNext()) {
2829        final MergePolicy.OneMerge merge = (MergePolicy.OneMerge) it.next();
2830        merge.optimize = true;
2831        merge.maxNumSegmentsOptimize = maxNumSegments;
2832      }
2833
2834      it = runningMerges.iterator();
2835      while(it.hasNext()) {
2836        final MergePolicy.OneMerge merge = (MergePolicy.OneMerge) it.next();
2837        merge.optimize = true;
2838        merge.maxNumSegmentsOptimize = maxNumSegments;
2839      }
2840    }
2841
2842    maybeMerge(maxNumSegments, true);
2843
2844    if (doWait) {
2845      synchronized(this) {
2846        while(true) {
2847
2848          if (hitOOM) {
2849            throw new IllegalStateException("this writer hit an OutOfMemoryError; cannot complete optimize");
2850          }
2851
2852          if (mergeExceptions.size() > 0) {
2853            // Forward any exceptions in background merge
2854            // threads to the current thread:
2855            final int size = mergeExceptions.size();
2856            for(int i=0;i<size;i++) {
2857              final MergePolicy.OneMerge merge = (MergePolicy.OneMerge) mergeExceptions.get(0);
2858              if (merge.optimize) {
2859                IOException err = new IOException("background merge hit exception: " + merge.segString(directory));
2860                final Throwable t = merge.getException();
2861                if (t != null)
2862                  err.initCause(t);
2863                throw err;
2864              }
2865            }
2866          }
2867
2868          if (optimizeMergesPending())
2869            doWait();
2870          else
2871            break;
2872        }
2873      }
2874
2875      // If close is called while we are still
2876      // running, throw an exception so the calling
2877      // thread will know the optimize did not
2878      // complete
2879      ensureOpen();
2880    }
2881
2882    // NOTE: in the ConcurrentMergeScheduler case, when
2883    // doWait is false, we can return immediately while
2884    // background threads accomplish the optimization
2885  }

3778            if (registerMerge(newMerge)) {
3779              merge = newMerge;

3781              // If this segment is not currently being
3782              // merged, then advance it to running & run
3783              // the merge ourself (below):
3784              pendingMerges.remove(merge);
3785              runningMerges.add(merge);
3786              break;
3787            }
3788          }
3789        }
3790
3791        if (!done && merge == null)
3792          // We are not yet done (external segments still
3793          // exist in segmentInfos), yet, all such segments
3794          // are currently "covered" by a pending or running
3795          // merge.  We now try to grab any pending merge
3796          // that involves external segments:
3797          merge = getNextExternalMerge();

3799        if (!done && merge == null)
3800          // We are not yet done, and, all external segments
3801          // fall under merges that the merge scheduler is
3802          // currently running.  So, we now wait and check
3803          // back to see if the merge has completed.
3804          doWait();
3805      }
3806
3807      if (merge != null) {
3808        any = true;
3809        merge(merge);
3810      }
3811    }

3815      // more merges may become necessary:

3968            } else {

3970            }
3971          }
3972        }
3973      }
3974    } catch (OutOfMemoryError oom) {

4447    // started merging:
4448    int docUpto = 0;
4449    int delCount = 0;
4450
4451    for(int i=0; i < sourceSegments.size(); i++) {
4452      SegmentInfo info = sourceSegments.info(i);
4453      int docCount = info.docCount;
4454      SegmentReader previousReader = merge.readersClone[i];
4455      SegmentReader currentReader = merge.readers[i];
4456      if (previousReader.hasDeletions()) {
4457
4458        // There were deletes on this segment when the merge
4459        // started.  The merge has collapsed away those
4460        // deletes, but, if new deletes were flushed since
4461        // the merge started, we must now carefully keep any
4462        // newly flushed deletes but mapping them to the new
4463        // docIDs.
4464
4465        if (currentReader.numDeletedDocs() > previousReader.numDeletedDocs()) {
4466          // This means this segment has had new deletes
4467          // committed since we started the merge, so we
4468          // must merge them:
4469          for(int j=0;j<docCount;j++) {
4470            if (previousReader.isDeleted(j))
4471              assert currentReader.isDeleted(j);
4472            else {
4473              if (currentReader.isDeleted(j)) {
4474                mergeReader.doDelete(docUpto);
4475                delCount++;
4476              }
4477              docUpto++;
4478            }
4479          }
4480        } else {
4481          docUpto += docCount - previousReader.numDeletedDocs();
4482        }
4483      } else if (currentReader.hasDeletions()) {
4484        // This segment had no deletes before but now it
4485        // does:
4486        for(int j=0; j<docCount; j++) {
4487          if (currentReader.isDeleted(j)) {
4488            mergeReader.doDelete(docUpto);
4489            delCount++;
4490          }
4491          docUpto++;
4492        }
4493      } else
4494        // No deletes before or after
4495        docUpto += info.docCount;
4496    }
4497
4498    assert mergeReader.numDeletedDocs() == delCount;
4499
4500    mergeReader.hasChanges = delCount >= 0;
4501  }

4727  final synchronized private void _mergeInit(MergePolicy.OneMerge merge) throws IOException {
4728
4729    assert testPoint("startMergeInit");
4730
4731    assert merge.registerDone;
4732    assert !merge.optimize || merge.maxNumSegmentsOptimize > 0;
4733
4734    if (hitOOM) {
4735      throw new IllegalStateException("this writer hit an OutOfMemoryError; cannot merge");
4736    }
4737
4738    if (merge.info != null)
4739      // mergeInit already done
4740      return;
4741
4742    if (merge.isAborted())
4743      return;
4744
4745    boolean changed = applyDeletes();
4746
4747    // If autoCommit == true then all deletes should have
4748    // been flushed when we flushed the last segment
4749    assert !changed || !autoCommit;
4750
4751    final SegmentInfos sourceSegments = merge.segments;
4752    final int end = sourceSegments.size();
4753
4754    // Check whether this merge will allow us to skip
4755    // merging the doc stores (stored field & vectors).
4756    // This is a very substantial optimization (saves tons
4757    // of IO) that can only be applied with
4758    // autoCommit=false.
4759
4760    Directory lastDir = directory;
4761    String lastDocStoreSegment = null;
4762    int next = -1;
4763
4764    boolean mergeDocStores = false;
4765    boolean doFlushDocStore = false;
4766    final String currentDocStoreSegment = docWriter.getDocStoreSegment();
4767
4768    // Test each segment to be merged: check if we need to
4769    // flush/merge doc stores
4770    for (int i = 0; i < end; i++) {
4771      SegmentInfo si = sourceSegments.info(i);
4772
4773      // If it has deletions we must merge the doc stores
4774      if (si.hasDeletions())
4775        mergeDocStores = true;
4776
4777      // If it has its own (private) doc stores we must
4778      // merge the doc stores
4779      if (-1 == si.getDocStoreOffset())
4780        mergeDocStores = true;
4781
4782      // If it has a different doc store segment than
4783      // previous segments, we must merge the doc stores
4784      String docStoreSegment = si.getDocStoreSegment();
4785      if (docStoreSegment == null)
4786        mergeDocStores = true;
4787      else if (lastDocStoreSegment == null)
4788        lastDocStoreSegment = docStoreSegment;
4789      else if (!lastDocStoreSegment.equals(docStoreSegment))
4790        mergeDocStores = true;
4791
4792      // Segments' docScoreOffsets must be in-order,
4793      // contiguous.  For the default merge policy now
4794      // this will always be the case but for an arbitrary
4795      // merge policy this may not be the case
4796      if (-1 == next)
4797        next = si.getDocStoreOffset() + si.docCount;
4798      else if (next != si.getDocStoreOffset())
4799        mergeDocStores = true;
4800      else
4801        next = si.getDocStoreOffset() + si.docCount;
4802      
4803      // If the segment comes from a different directory
4804      // we must merge
4805      if (lastDir != si.dir)
4806        mergeDocStores = true;
4807
4808      // If the segment is referencing the current "live"
4809      // doc store outputs then we must merge
4810      if (si.getDocStoreOffset() != -1 && currentDocStoreSegment != null && si.getDocStoreSegment().equals(currentDocStoreSegment)) {
4811        doFlushDocStore = true;
4812      }
4813    }
4814
4815    final int docStoreOffset;
4816    final String docStoreSegment;
4817    final boolean docStoreIsCompoundFile;
4818
4819    if (mergeDocStores) {
4820      docStoreOffset = -1;
4821      docStoreSegment = null;
4822      docStoreIsCompoundFile = false;
4823    } else {
4824      SegmentInfo si = sourceSegments.info(0);        
4825      docStoreOffset = si.getDocStoreOffset();
4826      docStoreSegment = si.getDocStoreSegment();
4827      docStoreIsCompoundFile = si.getDocStoreIsCompoundFile();
4828    }
4829
4830    if (mergeDocStores && doFlushDocStore) {
4831      // SegmentMerger intends to merge the doc stores
4832      // (stored fields, vectors), and at least one of the
4833      // segments to be merged refers to the currently
4834      // live doc stores.
4835
4836      // TODO: if we know we are about to merge away these
4837      // newly flushed doc store files then we should not
4838      // make compound file out of them...
4839      if (infoStream != null)
4840        message("now flush at merge");
4841      doFlush(true, false);
4842    }
4843
4844    merge.increfDone = true;
4845
4846    merge.mergeDocStores = mergeDocStores;
4847
4848    // Bind a new segment name here so even with
4849    // ConcurrentMergePolicy we keep deterministic segment
4850    // names.
4851    merge.info = new SegmentInfo(newSegmentName(), 0,
4852                                 directory, false, true,
4853                                 docStoreOffset,
4854                                 docStoreSegment,
4855                                 docStoreIsCompoundFile,
4856                                 false);
4857
4858
4859    Map details = new HashMap();
4860    details.put("optimize", merge.optimize+"");
4861    details.put("mergeFactor", end+"");
4862    details.put("mergeDocStores", mergeDocStores+"");
4863    setDiagnostics(merge.info, "merge", details);

4866    // this prevents it from getting selected for a merge
4867    // after our merge is done but while we are building the

5107    // referenced files in the new merge.info are incref'd
5108    // first:

5111    }

5117      // disk space used by the segments we just merged:

5119        final long size;
5120        synchronized(this) {
5121          size = merge.info.sizeInBytes();
5122        }
5123        commit(size);
5124      }
5125      
5126      success = false;
5127      final String compoundFileName = mergedName + "." + IndexFileNames.COMPOUND_FILE_EXTENSION;
5128
5129      try {
5130        merger.createCompoundFile(compoundFileName);
5131        success = true;
5132      } catch (IOException ioe) {

5134          if (merge.isAborted()) {
5135            // This can happen if rollback or close(false)
5136            // is called -- fall through to logic below to
5137            // remove the partially created CFS:
5138            success = true;
5139          } else

5141        }

5150          }
5151        }
5152      }
5153
5154      if (merge.isAborted()) {
5155        if (infoStream != null)
5156          message("abort merge after building CFS");
5157        deleter.deleteFile(compoundFileName);
5158        return 0;
5159      }
5160
5161      synchronized(this) {
5162        if (segmentInfos.indexOf(merge.info) == -1 || merge.isAborted()) {
5163          // Our segment (committed in non-compound
5164          // format) got merged away while we were
5165          // building the compound format.
5166          deleter.deleteFile(compoundFileName);
5167        } else {
5168          merge.info.setUseCompoundFile(true);
5169          checkpoint();
5170        }
5171      }
5172    }
5173
5174    // Force a sync after commiting the merge.  Once this
5175    // sync completes then all index files referenced by the
5176    // current segmentInfos are on stable storage so if the
5177    // OS/machine crashes, or power cord is yanked, the
5178    // index will be intact.  Note that this is just one
5179    // (somewhat arbitrary) policy; we could try other
5180    // policies like only sync if it's been > X minutes or
5181    // more than Y bytes have been written, etc.

5183      final long size;
5184      synchronized(this) {
5185        size = merge.info.sizeInBytes();
5186      }
5187      commit(size);
5188    }
5189

5377  /** Walk through all files referenced by the current
5378   *  segmentInfos and ask the Directory to sync each file,
5379   *  if it wasn't already.  If that succeeds, then we
5380   *  prepare a new segments_N file but do not fully commit
5381   *  it. */
5382  private void startCommit(long sizeInBytes, Map commitUserData) throws IOException {
5383
5384    assert testPoint("startStartCommit");
5385
5386    // TODO: as of LUCENE-2095, we can simplify this method,
5387    // since only 1 thread can be in here at once
5388
5389    if (hitOOM) {
5390      throw new IllegalStateException("this writer hit an OutOfMemoryError; cannot commit");
5391    }
5392
5393    try {
5394
5395      if (infoStream != null)
5396        message("startCommit(): start sizeInBytes=" + sizeInBytes);
5397
5398      if (sizeInBytes > 0)
5399        syncPause(sizeInBytes);
5400
5401      SegmentInfos toSync = null;
5402      final long myChangeCount;
5403
5404      synchronized(this) {
5405
5406        // sizeInBytes > 0 means this is an autoCommit at
5407        // the end of a merge.  If at this point stopMerges
5408        // is true (which means a rollback() or
5409        // rollbackTransaction() is waiting for us to
5410        // finish), we skip the commit to avoid deadlock
5411        if (sizeInBytes > 0 && stopMerges)
5412          return;
5413
5414        // Wait for any running addIndexes to complete
5415        // first, then block any from running until we've
5416        // copied the segmentInfos we intend to sync:
5417        blockAddIndexes(false);
5418
5419        // On commit the segmentInfos must never
5420        // reference a segment in another directory:
5421        assert !hasExternalSegments();
5422
5423        try {
5424
5425          assert lastCommitChangeCount <= changeCount;
5426
5427          if (changeCount == lastCommitChangeCount) {
5428            if (infoStream != null)
5429              message("  skip startCommit(): no changes pending");
5430            return;
5431          }
5432
5433          // First, we clone & incref the segmentInfos we intend
5434          // to sync, then, without locking, we sync() each file
5435          // referenced by toSync, in the background.  Multiple
5436          // threads can be doing this at once, if say a large
5437          // merge and a small merge finish at the same time:
5438
5439          if (infoStream != null)
5440            message("startCommit index=" + segString(segmentInfos) + " changeCount=" + changeCount);
5441          
5442          readerPool.commit();
5443          
5444          toSync = (SegmentInfos) segmentInfos.clone();
5445
5446          if (commitUserData != null)
5447            toSync.setUserData(commitUserData);
5448
5449          deleter.incRef(toSync, false);
5450          myChangeCount = changeCount;
5451
5452          Iterator it = toSync.files(directory, false).iterator();
5453          while(it.hasNext()) {
5454            String fileName = (String) it.next();
5455            assert directory.fileExists(fileName): "file " + fileName + " does not exist";
5456          }
5457
5458        } finally {
5459          resumeAddIndexes();
5460        }
5461      }
5462
5463      assert testPoint("midStartCommit");
5464
5465      boolean setPending = false;
5466
5467      try {
5468
5469        // Loop until all files toSync references are sync'd:
5470        while(true) {
5471
5472          final Collection pending = new ArrayList();
5473
5474          Iterator it = toSync.files(directory, false).iterator();
5475          while(it.hasNext()) {
5476            final String fileName = (String) it.next();

5478              boolean success = false;
5479              try {
5480                // Because we incRef'd this commit point, above,
5481                // the file had better exist:
5482                assert directory.fileExists(fileName): "file '" + fileName + "' does not exist dir=" + directory;
5483                if (infoStream != null)
5484                  message("now sync " + fileName);
5485                directory.sync(fileName);
5486                success = true;
5487              } finally {
5488                finishSync(fileName, success);
5489              }
5490            }
5491          }
5492
5493          // All files that I require are either synced or being
5494          // synced by other threads.  If they are being synced,
5495          // we must at this point block until they are done.
5496          // If this returns false, that means an error in
5497          // another thread resulted in failing to actually
5498          // sync one of our files, so we repeat:
5499          if (waitForAllSynced(pending))
5500            break;
5501        }
5502
5503        assert testPoint("midStartCommit2");
5504
5505        synchronized(this) {
5506          // If someone saved a newer version of segments file
5507          // since I first started syncing my version, I can
5508          // safely skip saving myself since I've been
5509          // superseded:
5510
5511          while(true) {
5512            if (myChangeCount <= lastCommitChangeCount) {
5513              if (infoStream != null) {
5514                message("sync superseded by newer infos");
5515              }
5516              break;
5517            } else if (pendingCommit == null) {
5518              // My turn to commit
5519
5520              if (segmentInfos.getGeneration() > toSync.getGeneration())
5521                toSync.updateGeneration(segmentInfos);
5522
5523              boolean success = false;
5524              try {
5525
5526                // Exception here means nothing is prepared
5527                // (this method unwinds everything it did on
5528                // an exception)

5530                  toSync.prepareCommit(directory);
5531                } finally {
5532                  // Have our master segmentInfos record the
5533                  // generations we just prepared.  We do this
5534                  // on error or success so we don't
5535                  // double-write a segments_N file.
5536                  segmentInfos.updateGeneration(toSync);
5537                }
5538
5539                assert pendingCommit == null;
5540                setPending = true;
5541                pendingCommit = toSync;
5542                pendingCommitChangeCount = myChangeCount;
5543                success = true;
5544              } finally {
5545                if (!success && infoStream != null)
5546                  message("hit exception committing segments file");
5547              }

5549            } else {
5550              // Must wait for other commit to complete
5551              doWait();
5552            }
5553          }
5554        }
5555

5557          message("done all syncs");

5559        assert testPoint("midStartCommitSuccess");
5560
5561      } finally {

5566      }
